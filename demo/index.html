SOCIAL MINUS SPYING

social sharing buttons that respect your damn privacy

QUICK START


1) Download S-S.js, and embed it anywhere on your web page.
<script src="S-S.js"></script>

2) Insert the following snippet where you want your social buttons:
<div class="s-s" data-type="facebook"></div>
<div class="s-s" data-type="twitter"></div>

3) That's it! You'll get privacy-respecting share buttons like these:
They have the same functionality as the official social sharing widgets, but they don't use cookies, don't load external iframes, and in fact never directly call the Facebook/Twitter servers. Not only are they more customizeable and more lightweight, these buttons give a damn about Your Damn Privacy™.

HOW THEY PROTECT PRIVACY

HOW TO CUSTOMIZE


1) Customizing links, labels, and share description.

You can change the text and links of a share button simply by adding data- attributes to the HTML. For example, this code...
<div class="s-s"
     data-type="twitter"
     data-link="http://www.catlateraldamage.com/"
     data-label="Talk about a CAT-ASTROPHE"
     data-text="A first-person destructive cat simulator">
</div>
... will give you this button.



2) Customizing styles.

Because the share button is not inside an external iframe, you can directly modify its look simply by including a stylesheet! (protip: give the buttons id's to change their styles individually)
<style>
#please_dont_do_this{
     font-family:"Comic Sans MS";
     border-radius: 50px;
}
#please_dont_do_this #ss_label{
     background: #F9429E;
}
</style>
<div id="please_dont_do_this" class="s-s" data-type="facebook"></div>
which gives you... uh... this.



You can also pass in an HTML template as a data- attribute. Here's a template for a very minimalist Twitter Share button. (highlighted in the code for covenience)
<div class="s-s" data-template="<a target='_blank' href='https://twitter.com/share?url={{link}}&text={{text}}'>tweet</a> ({{count}})"> data-type="twitter" </div>

3) Customizing APIs. (advanced. you'll likely never need this)

Whether it's requesting different stats, or building a social share button for a completely different site, or making a button that calls an API that only returns the number "8"... you can easily extend S-S to fit your needs.

To show this in action, a button that calls an API that only returns the number "8". The "API" in question is just a JSON file I posted on Pastebin. To change the API's called, I create a new config object in the global object, SS_SOCIAL_API, like so:
<script>
window.SS_SOCIAL_API.stanley = {
     template: "<button style='font-size:30px' onclick='alert({{count}})'>{{label}}</button>",
     label: "The Stanley Parable Demo",
     requestURI: "http://pastebin.com/raw.php?i=eBHzdqUP&throwaway=",
     requestProperty: "count"
};
</script>
<div class="s-s" data-type="stanley"></div>
